## What

`exampleTargetDependenciesConstantExcludesTestServer()` in `Tests/FoundationModelsMCPTests/PackageDependencyTests.swift` is meant to guard against `"MCPTestServer"` being added to the shared `exampleTargetDependencies` constant in `Package.swift` (a regression the per-target `exampleTargetExcludesTestServer(targetName:)` check can't catch, since the four generated targets reference the constant by name rather than spelling out dependencies inline).

The implementation is vacuous. It does:

```swift
guard let constantStart = source.range(of: "let exampleTargetDependencies"),
    let closingBracket = source[constantStart.upperBound...].range(of: "]")
```

`Package.swift` declares the constant as:

```swift
let exampleTargetDependencies: [Target.Dependency] = [
    "FoundationModelsMCP",
    "ExampleSupport",
    mcpProduct,
]
```

The *first* `"]"` after the constant name closes the **type annotation** `[Target.Dependency]`, not the array literal. So `constantDeclaration` only ever captures `"let exampleTargetDependencies: [Target.Dependency]"` — the actual array body is never inspected. `#expect(!constantDeclaration.contains("MCPTestServer"))` passes unconditionally no matter what is in the real dependency list, including `"MCPTestServer"` itself. The regression protection the test's own doc comment claims to provide ("This checks the constant's own declaration directly, closing that gap") does not exist.

The sibling test `exampleTargetSpecsDeclaresExpectedNames()` in the same file avoids this exact trap by searching for `"\n]"` (a closing bracket at the start of its own line) instead of a bare `"]"` — the correct pattern is already known and used elsewhere in the file, just not applied here.

## Fix

Change the closing-bracket search in `exampleTargetDependenciesConstantExcludesTestServer()` to anchor past the type annotation — e.g. search for `" = ["` (the assignment) or `"\n]"` (matching the sibling test's pattern) instead of a bare `"]"` right after the constant name.
