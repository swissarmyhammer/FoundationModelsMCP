## What
Create the first four `Examples/` executable targets per plan.md Examples §1–4, each runnable via `swift run <Name>` and compiled in CI. **Toy servers come from the `MCPTestServer` utility target** (spawned as a stdio subprocess in the matching mode — echo mode for EchoTool, filesystem-style multi-tool mode for FileAssistant); examples never import the test target. Targets: **EchoTool** (~20-line hello world: spawn MCPTestServer echo → MCPServer → LanguageModelSession(mcp:) on the system model → one prompt, one tool call); **FileAssistant** (multi-tool mode, model selects among tools, isError bubbling on a missing-file prompt); **ToolPicking** (one loose MCPTool + a native Swift Tool composed in one session — MCPToolProvider flattening); **RemoteHTTP** (HTTPClientTransport + host-supplied bearer token — the delegated-auth decision).
